# 🔐 Security Policy

Thank you for taking an interest in the security of **OSINT Tool**.  
I take security seriously and appreciate the responsible disclosure of any vulnerabilities.

---

## ⭐ Supported Versions

Only the latest stable releases of this tool receive security updates.

| Version        | Supported |
|----------------|-----------|
| v2.x.x         | ✅ Active |
| v1.x.x         | ⚠ Limited |
| < v1.0         | ❌ Not Supported |

If you’re using an older version, you are strongly encouraged to update to the latest release.

---

## 🚨 Reporting a Vulnerability

If you discover a security issue, please report it responsibly.

### 📩 How to Report
Send an email to:

**📧 Anish_Kushwaha@proton.me**  
Subject line: **"[Security Report] Vulnerability in Social Media Finder OSINT"**

### ✔ What to Include
When reporting a vulnerability, please include:

- A detailed description of the issue  
- Steps to reproduce  
- Screenshots or proof-of-concept (if possible)  
- Expected vs actual behavior  
- Potential impact  
- Any suggested fix (optional)

### 📆 Response Time
- You will receive an acknowledgment within **48 hours**
- A full response or fix will usually be provided within **3–7 days** depending on severity

---

## 🛡 Responsible Disclosure Guidelines

To keep users safe, please follow these rules:

- Do **not** publicly disclose the vulnerability until an official fix is released  
- Do **not** use the vulnerability for malicious purposes  
- Do **not** test against systems you do not own or have permission for  

---

## 🔒 Security Practices Followed

The project follows:

- Minimal external API usage  
- No collection of personal or sensitive data  
- Open-source transparency  
- Regular dependency updates  
- Community-driven vulnerability reporting  

---

Your contribution helps keep this project safe for everyone.
